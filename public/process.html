Process Notes

A look behind the curtain at the problem-solving workflow used to build this site.

Key Iterations

Serverless API Configuration Backend

Challenge: The chatbot returned a 404 error on Vercel deployment, despite working locally.

"I made all the changes you said... but the chatbot isn't working (unable to connect to database)."

Solution: Identified that Vercel wasn't detecting the backend function due to incorrect Root Directory settings. Updated project configuration to point to the correct root and added a package.json with type: module to support modern ES6 import syntax.

Background Visuals Pivot CSS

Challenge: Complex SVG shapes were inconsistent and difficult to style dynamically.

"The blobs are still not shoe shaped... let's change the blobs to just actual blobs? random rough circle shapes."

Solution: Pivoted from rigid SVG paths to CSS-styled div elements. Used border-radius with percentage values and @keyframes to create a "Lava Lamp" morphing effect that is smoother and more performant.

Chat Input Security Logic

Challenge: Sample question button broke the JavaScript functionality due to unescaped characters.

"The 'beginner advice' button in the chatbot does not work."

Solution: Identified that the apostrophe in "I'm" was terminating the JS string early. Sanitized the HTML attribute by using double backslashes or removing the contraction entirely to ensure valid syntax.

DeepSeek Integration AI

Challenge: Needed a fast, cost-effective LLM API with strict inventory constraints.

"I want to create a sample e-commerce website... using an API from an LLM... is Deepseek an option?"

Solution: Integrated DeepSeek Chat API. Engineered the system prompt to act as a "constraint engine," forcing the model to only recommend products from a provided JSON-like inventory list, preventing hallucinations.

Workflow Strategies

Iterative Debugging

When the Vercel deployment failed, we didn't rewrite the whole code. We isolated the issue (Build Logs vs. Function Logs) and confirmed success by verifying the "Functions" tab appeared in the deployment summary before moving on.

Visual Prototyping

Instead of guessing shapes, we used an SVG converter tool to try specific silhouettes. When that didn't fit the aesthetic, we quickly pivoted to a pure CSS solution to match the "Dark Theme" vibe perfectly.

Security-First Architecture

Recognized that exposing API keys on a static GitHub Pages site is a security risk. Implemented a "Serverless Proxy" pattern using Vercel Functions to keep secrets secure on the backend while keeping the frontend lightweight.
