Process Notes

A look behind the curtain at the problem-solving workflow used to build this site.

Key Iterations

RAG Implementation Backend

Challenge: The AI hallucinated shoe specs because it relied on general training data, not store inventory.

"The bot is confused about both the inventory and IDing incorrectly... we need a separate logic."

Solution: Implemented Retrieval-Augmented Generation (RAG). Built a structured JSON dataset of 8 shoes with specific fields (Weight, Type, Price). Created a retrieveContext() function to filter this data before sending it to the LLM.

API Error Handling Frontend

Challenge: Chatbot hung indefinitely ("Thinking...") when server errors occurred.

"The bot hangs indefinitely and doesn't respond... TypeError: Cannot set properties of null."

Solution: Added robust try/catch blocks and improved frontend response checking to handle HTTP errors (like 402 Payment Required) gracefully instead of failing silently.

Module Import Debugging Ops

Challenge: Vercel serverless functions crashed silently when importing local JSON files.

"I no longer get the server error, but... the bot hangs indefinitely."

Solution: Identified that import data from '../data.json' fails in the specific Vercel runtime environment. Hardcoded the data object directly into the function file to guarantee availability.

Vision Feature Pivot Strategy

Challenge: External Vision APIs (HuggingFace, Google Gemini) were unreliable or required complex authentication that failed in the portfolio context.

"This vision thing is proving too complicated, let's scrap the vision function..."

Solution: Pivoted to a pure Text-Based RAG system. This focused the project on data accuracy and logical querying, which are higher-value skills for backend engineering roles.

Workflow Strategies

Structured Data Engineering

Instead of relying on the LLM to "know" facts, we shifted to a "Grounding" strategy. By maintaining a clean data.json schema, we ensure the AI's answers are always factually correct regarding price and specs.

Debugging via Logs

Used Vercel's Realtime Logs to diagnose API failures. This allowed us to distinguish between "Network Errors," "Authentication Errors," and "Runtime Crashes," significantly speeding up the debugging loop.

Fail-Fast Mentality

When the Vision API integration proved unstable, we quickly evaluated the ROI and pivoted to a Text-RAG system. This ensured the core portfolio value (backend logic) wasn't compromised by a flaky 3rd party dependency.
