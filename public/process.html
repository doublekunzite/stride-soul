AI Integration & Chatbot Features

Context-Aware RAG Backend

Challenge: The bot was reciting shoe weights (e.g., "303g") in inappropriate contexts like general comfort questions.

"It can mention the weights if asked, but it should behave more naturally... not even mention the price."

Solution: Implemented "Intent-Based Retrieval." The system now filters data before sending it to the LLM. If the user asks about "comfort," we strip price/weight data from the context window, forcing the AI to focus purely on descriptive text.

Competitor Knowledge Logic Data

Challenge: The bot was hallucinating inventory when asked about competitor shoes (e.g., Nike Pegasus), but we wanted it to be knowledgeable yet sales-focused.

"Is this going to be an ongoing issue... or are there possible workarounds here?"

Solution: Built a "Logic Tree" in the backend. If a competitor is mentioned, the system triggers a specific protocol: 1. Acknowledge the competitor shoe using market knowledge. 2. Clarify it's not in stock. 3. Pivot to a recommendation from our inventory.

Persona Engineering Prompting

Challenge: The AI was using robotic labels like "PIVOT:" in its responses, breaking the user immersion.

"Adjust the prompting so that it doesn't write 'acknowledge' etc and speaks in a natural, casual tone."

Solution: Refined the System Prompt to enforce a "Natural Logic." The AI is instructed to follow the logic steps *silently* and output only conversational text, resulting in a helpful salesperson tone rather than a robotic flowchart.

Serverless API Configuration Backend

Challenge: The chatbot returned a 404 error on Vercel deployment, despite working locally.

"I made all the changes you said... but the chatbot isn't working."

Solution: Identified that Vercel wasn't detecting the backend function due to incorrect Root Directory settings. Updated project configuration and added package.json with type: module to support modern ES6 syntax.

Design & Technical Engineering

Glassmorphism UI Frontend

Challenge: The chat window looked "rudimentary" and flat against the dark background.

"Can we polish this up... glassy appearance, and something that gives it a bit of 3d texture so it pops?"

Solution: Applied CSS backdrop-filter: blur() for a frosted glass effect, layered shadows for 3D depth, and designed a "Tech Stack" badge to professionally display the model version (DeepSeek V4 Flash).

Background Visuals Pivot CSS

Challenge: Complex SVG shapes were inconsistent and difficult to style dynamically.

"The blobs are still not shoe shaped... let's change the blobs to just actual blobs."

Solution: Pivoted from rigid SVG paths to CSS-styled div elements. Used border-radius with percentage values and @keyframes to create a "Lava Lamp" morphing effect that is smoother and more performant.

Module Import Debugging Ops

Challenge: Vercel serverless functions crashed silently when importing local JSON files.

"I no longer get the server error, but... the bot hangs indefinitely."

Solution: Identified that import data from '../data.json' fails in the specific Vercel runtime environment. Hardcoded the data object directly into the function file to guarantee availability.

Sample Prompts

Diagnostic Analysis Workflow
"The bot is both confused about the inventory and IDing incorrectly. don't change the code further yet; what do you think could be happening here and what are some possible solutions?"

Analysis: Instead of just reporting a bug, I asked for a root cause analysis. By instructing the AI not to change code yet, I forced it to explain the problem first, which helped me understand the distinction between "Knowledge" and "Inventory" in the RAG system.

Behavioral Specification Prompting
"It can mention the weights if asked, but it should behave more naturally... not even mention the price, just the model, if asked about flat feet for example. also... let's again make it more like an actual sales person."

Analysis: I used conditional logic in my prompt ("if asked... then...") to guide the AI's behavior. This goes beyond simple feature requests by defining the context in which specific data should be used, effectively "programming" the persona's social skills.

Logic Isolation Architecture
"Can we restrict that new logic [for questions about competing brands] to this specific question type... like a logic tree, if [question about outside shoe] then [logic]. Then we can avoid contaminating the other types of questions."

Analysis: I recognized that a global prompt change would hurt existing functionality. I proposed a conditional logic flow (Logic Tree) to isolate the risk. This is a senior-level architectural decision that prevents "feature creep" from breaking core features.

Workflow Strategies

Prompt Engineering

Rather than just asking the AI to implement a solution, I often tell it to hold off making changes and first give me options. I use detailed, descriptive prompts to minimize guessing, but I also know when to give the AI leeway to be "creative" during brainstorming phases.

Structured Data Engineering

Instead of relying on the chatbot to "know" facts, I used a "Grounding" strategy. By maintaining a clean JSON schema (a backend data file for the chatbot), we ensure the AI's answers are always factually correct regarding price and specs.

Creative Problem Solving

I actively guide the AI through its blind spots. For example, knowing LLMs struggle with hallucination when providing URLs, I'll have the AI put a placeholder in the code and retrieve the correct URL myself, effectively manual-fixing the "last mile" of the code.
