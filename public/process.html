AI & Backend Architecture

Context-Aware RAG Backend

Challenge: The bot was reciting shoe weights (e.g., "303g") in inappropriate contexts like general comfort questions.

"It can mention the weights if asked, but it should behave more naturally... not even mention the price."

Solution: Implemented "Intent-Based Retrieval." The system now filters data before sending it to the LLM. If the user asks about "comfort," we strip price/weight data from the context window, forcing the AI to focus purely on descriptive text.

Competitor Knowledge Logic Data

Challenge: The bot was hallucinating inventory when asked about competitor shoes (e.g., Nike Pegasus), but we wanted it to be knowledgeable yet sales-focused.

"Is this going to be an ongoing issue... or are there possible workarounds here?"

Solution: Built a "Logic Tree" in the backend. If a competitor is mentioned, the system triggers a specific protocol: 1. Acknowledge the competitor shoe using market knowledge. 2. Clarify it's not in stock. 3. Pivot to a recommendation of something similar from our inventory.

Frontend Engineering & UX

Dynamic State Management Frontend Logic

Challenge: Users needed to add multiple quantities of the same shoe, but the cart was creating duplicate entries, making the total calculation incorrect.

Solution: Refactored the cart logic to use unique keys (Product ID + Size). The system now checks for existing items to increment quantity rather than pushing duplicates, ensuring accurate subtotals and a cleaner UI.

Feedback Loop UI UX

Challenge: Forms often fail silently or look unprofessional without clear user feedback.

Solution: Implemented a robust validation flow for the newsletter. If the email is invalid, the UI reacts with a "shake" animation and error styling. On success, a "Toast" notification slides in, confirming the action without intrusive alerts.

Glassmorphism UI Frontend

Challenge: The chat window looked "rudimentary" and flat against the dark background.

"Can we polish this up... glassy appearance, and something that gives it a bit of 3d texture so it pops?"

Solution: Applied CSS backdrop-filter: blur() for a frosted glass effect, layered shadows for 3D depth, and designed a "Tech Stack" badge to professionally display the model version.

Workflow Strategies

Prompt Engineering

I use a "Diagnostic First" approach. Instead of asking the AI to implement a solution immediately, I ask it to hold off and provide a root cause analysis. This forces the AI to explain the problem, ensuring I understand the logic before code is generated.

Structured Data Grounding

To prevent AI hallucination, I rely on "Grounding." By maintaining a clean JSON schema (a backend data file) separate from the AI's training data, we ensure the chatbot's answers are always factually correct regarding price and specs.
